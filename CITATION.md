# Как цитировать EIS Solver

Если программа использовалась для загрузки данных, проверки Крамерса—Кронига,
подбора эквивалентной схемы, построения графиков или инженерного экспорта,
укажите EIS Solver в разделе методов и добавьте программную ссылку в список
литературы.

## Минимальная информация в статье

Укажите:

1. название **EIS Solver**;
2. точную версию или Git commit;
3. ссылку на репозиторий;
4. режим подбора (`adaptive_v2`, именованный preset или ручная схема);
5. использованную частотную область и статус Lin-KK;
6. выбранную схему и существенные настройки оптимизатора, если от них зависит
   научный вывод.

Пример для раздела Methods:

> Спектры анализировали в EIS Solver v0.9.0 (Stepan Tarasov, 2026) с
> автоматической маршрутизацией adaptive_v2. Согласованность данных проверяли
> методом Lin-KK; модели сравнивали по BIC. Для воспроизводимости сохраняли
> полный экспорт результатов и диагностические флаги.

Не называйте `OK` доказательством физической истинности схемы. Если вывод
основан на конкретных методах или сторонних данных, отдельно цитируйте
первоисточники этих методов, библиотек и наборов данных. Ссылка на EIS Solver
их не заменяет.

## Рекомендуемая программная ссылка

**Текстовый вариант:**

> Tarasov, S. (2026). *EIS Solver* (Version 0.9.0) [Computer software].
> https://github.com/stpntrsvv/eis

После появления архивного DOI релиза ссылку на GitHub следует заменить или
дополнить DOI. Для другой версии замените номер версии и год выпуска.

## BibTeX

```bibtex
@software{tarasov_eis_solver_2026,
  author  = {Tarasov, Stepan},
  title   = {EIS Solver},
  year    = {2026},
  version = {0.9.0},
  url     = {https://github.com/stpntrsvv/eis},
  note    = {Electrochemical impedance spectroscopy analysis software}
}
```

## Если цитируется конкретный результат

Для воспроизводимой ссылки дополнительно укажите:

- SHA Git commit или тег релиза;
- имя входного набора данных и его DOI/источник;
- экспортированный JSON/CSV-паспорт анализа;
- ручную схему, bounds, число рестартов и seed, если они задавались;
- версию ngspice для SPICE-экспорта либо параметры дискретизации для
  контроллерного C-пакета.

Машиночитаемые метаданные находятся в `CITATION.cff`. При выпуске новой версии
номер и дата одновременно обновляются в `eis_version.py`, `CITATION.cff` и
релизных заметках.
