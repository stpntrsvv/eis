# Validation data workspace

Эта папка хранит воспроизводимый контур проверки EIS Solver на открытых экспериментальных данных.

- `manifest.json` — источники, версии, лицензии и назначение корпусов;
- `raw/` — загруженные исходники, исключённые из Git из-за размера;
- `artifacts/` — машинные результаты CLI, также исключённые из Git;
- `manifests/` — замороженные договоры воспроизводимых проверок;
- `reports/` — компактные отчёты и выводы, которые можно хранить в репозитории.

Сырые данные следует получать только по ссылкам из manifest. При публикации результатов необходимо указывать авторов и DOI исходного набора.

Инженерный корпус SPICE v1 задан файлом
`manifests/spice_engineering_corpus_v1.json`; итоговый отчёт находится в
`reports/2026-07-19-spice-engineering-corpus-v1.md`.

Корпус обусловленности SPICE v2 задан файлом
`manifests/spice_conditioning_corpus_v2.json`; отчёт находится в
`reports/2026-07-19-spice-conditioning-corpus-v2.md`.

Первый пользовательский экспорт через командную строку записан в отчёте
`reports/2026-07-19-spice-user-export-smoke.md`.

Первая сквозная проверка экспорта C для контроллера описана в
`reports/2026-07-19-controller-c-export-smoke.md`. Проверены оба варианта
(`float32` и Q31), строгая сборка C99 и побитовое совпадение Q31 с
целочисленным имитатором. Целевая проверка на STM32F103 ещё не выполнена.

Приёмка кандидата EIS Solver `0.9.0`, унификация GUI/CLI, функциональная
проверка собранного exe и лицензионные ограничения записаны в
`reports/2026-07-19-v0.9.0-release-candidate.md`.
