# Первый пользовательский экспорт SPICE

Дата: 2026-07-19.

## Проверенный путь

Команда:

```powershell
.\.venv\Scripts\python.exe eis_cli.py "double very good eis.txt" `
  --circuit "R0-p(R1,CPE0)-p(R2,CPE1)" `
  --spice-export validation_data\artifacts\spice_user_export_smoke `
  --ngspice D:\eis2\.runtime\ngspice-46\Spice64\bin\ngspice_con.exe `
  --format json `
  --output validation_data\artifacts\spice_user_export_smoke_analysis.json `
  --quiet
```

Команда завершилась с кодом `0` и атомарно создала каталог:

```text
spice_user_export_smoke/
  model.lib
  passport.json
```

## Договор безопасности

- экспорт разрешён только для одного входного файла;
- режим анализа обязан быть полным;
- научная модель должна иметь статус `OK` или `WARN`;
- проверка Крамерса — Кронига должна иметь статус `PASS` или `WARN`;
- ни один внутренний или внешний порог не обходится;
- отсутствие ngspice считается отказом;
- существующий каталог пакета не перезаписывается;
- временный пакет становится видимым только после полной записи
  `model.lib` и строгого JSON-паспорта;
- нечисловые диагностические значения записываются как `null`, а не как
  нестандартные значения JSON.

Проверочный пакет находится в исключённом из Git каталоге
`validation_data/artifacts/spice_user_export_smoke/`.

## Подключение к графическому интерфейсу

Тот же экспортёр подключён к действию `Пакет SPICE...` в конце основной
панели инструментов. Действие и его разделитель полностью скрыты вне Pro mode.
Отдельного пункта главного меню и кнопки в рабочей области нет.

Три внеэкранных Qt-теста проверяют:

- скрытие и появление действия вместе с Pro mode;
- передачу выбранного спектра в общий экспортёр;
- показ отказа и восстановление доступности действия.
