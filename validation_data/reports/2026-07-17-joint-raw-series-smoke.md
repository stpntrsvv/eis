# Совместный fit сырых EIS-серий: первый сквозной прогон

Дата: 2026-07-17.

Добавлен отдельный модуль `eis_joint.py`. Он намеренно не включён в обычный анализ одиночного спектра и запускается только с явным CSV-manifest, содержащим как минимум `file` и `soc`.

В одной целевой функции находятся комплексные нормированные остатки всех спектров и регуляризатор изменения наклона параметрических траекторий по SOC. Положительные параметры оптимизируются в логарифмическом масштабе. Перед совместной оптимизацией обычный независимый multi-start даёт начальные значения и сохраняется в результате для сравнения.

Дымовой прогон выполнен на пяти сырых спектрах заряда LG MJ1, SOC 0–4%, со схемой `R0-p(R1,CPE0)`. Оптимизатор успешно завершился за 14 вычислений целевой функции, совместный результат содержит параметры всех пяти спектров. Это проверка работоспособности контура, не валидация физической модели и не основание выбирать данную схему.

```powershell
.\.venv\Scripts\python.exe eis_joint.py validation_data\manifests\lg_mj1_charge_smoke.csv `
  --circuit "R0-p(R1,CPE0)" --smoothness 1 --max-evaluations 1000 --restarts 1 `
  --output validation_data\artifacts\joint_smoke\result.json
```

Следующая проверка должна сравнить несколько значений регуляризации через удерживаемые SOC-точки, а затем выполнить bootstrap по спектрам. Иначе коэффициент гладкости останется ручной ручкой, способной скрыть реальный переход.
