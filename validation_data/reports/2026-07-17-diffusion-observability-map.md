# Карта наблюдаемости диффузионного семейства

## Постановка

Предыдущий случайный synthetic benchmark показал, что bootstrap может быть
стабильно неправ. Новая карта управляет двумя физически важными осями:

- RMS-вклад диффузионного элемента относительно базового импеданса:
  `1%`, `5%`, `20%`;
- шум: `0,5%`, `2%`.

Для `Wo/Ws` дополнительно меняется положение характерной частоты: ниже
низкочастотного края, внутри диапазона и в верхней части области, разрешённой
production bounds. Все синтетические параметры находятся внутри пространства
fit.

## Реализация

Добавлен `eis_diffusion_map.py`. Сила `W/Wo/Ws` вычисляется так, чтобы
фактический RMS импедансного вклада совпадал с заказанной долей. Manifest
сохраняет:

```text
diffusion_kind
signal_fraction_requested / achieved
characteristic_position / frequency
noise_fraction
signal_to_noise_ratio
```

Полная дешёвая сетка содержит 42 спектра. Fit четырёх кандидатов занял около
21 секунды. Bootstrap запускается только на выбранных ячейках.

## Fit-карта

| Метрика | Результат |
|---|---:|
| Завершено | 42/42 |
| Семейство определено верно | 28/42 |
| Точная топология верна | 7/42 |
| Истина вошла в BIC-окно | 10/42 |

По типам:

| Тип и положение | Семейство | Топология | Истина в BIC-окне |
|---|---:|---:|---:|
| `W` | 5/6 | 5/6 | 6/6 |
| `Wo`, ниже края | 6/6 | 1/6 | 1/6 |
| `Wo`, внутри | 6/6 | 0/6 | 0/6 |
| `Wo`, верхняя область | 5/6 | 1/6 | 1/6 |
| `Ws`, ниже края | 6/6 | 0/6 | 2/6 |
| `Ws`, внутри | 0/6 | 0/6 | 0/6 |
| `Ws`, верхняя область | 0/6 | 0/6 | 0/6 |

Глобальный signal-to-noise не определяет разрешимость: даже при RMS-вкладе
20% и шуме 0,5% `Ws` может полностью поглощаться базовой схемой, а `Wo/Ws`
часто аппроксимируются `W`.

## Bootstrap сильных ячеек

Проверены семь ячеек с вкладом 20%, шумом 0,5% и 10 bootstrap-повторами.
При обычном 90%-пороге получено шесть точных рекомендаций, четыре из них
ложные. Bootstrap закрепляет выбранное локальное объяснение и не восстанавливает
истинное граничное условие.

## Экспериментальный gate

Добавлен только в benchmark, не в production inference:

```text
winner_family == inductive_diffusion
AND family bootstrap >= 90%
AND ΔBIC(diffusion против base) >= 10
→ recommended_family = inductive_diffusion
→ recommended_topology = null
```

Если gate не пройден, результат — `insufficient_information`; победа базовой
схемы не интерпретируется как доказанное отсутствие диффузии.

На семи сильных ячейках и четырёх отрицательных stress-контролях:

- семейство рекомендовано 4 раза;
- все 4 рекомендации правильны;
- 7 честных отказов;
- 0 ложноположительных диффузионных рекомендаций;
- точная топология не рекомендовалась.

Это pilot, а не окончательная калибровка. Перед production-интеграцией нужны
повторы по seeds и более плотная сетка около границы.

## Вывод

Карта решает инженерную часть вопроса асимметрично:

- сильное положительное свидетельство позволяет рекомендовать диффузионное
  ECM-семейство;
- отрицательный результат не доказывает отсутствие слабой диффузии;
- `W/Wo/Ws` по текущей калибровке рекомендовать нельзя.
