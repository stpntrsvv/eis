# Проверка multi-start фитинга — 2026-07-17

## Постановка

Предыдущий синтетический benchmark восстановил только `2/10` двухдуговых топологий. Чтобы отделить проблему локального минимума от проблемы идентифицируемости и ранжирования, те же десять CSV были повторно обработаны без изменения шума, частот, схем и параметров.

Сравнивались один геометрический старт и четыре детерминированных старта на каждую схему. Использовались одинаковый seed `20260719`, лимит 5000 вычислений на старт, семейство `simple` и политика выбора `ΔBIC ≤ 2`.

## Реализация

`fit_circuit()` теперь умеет:

1. сохранить исходное геометрическое приближение первым стартом;
2. создать дополнительные ограниченные логарифмические возмущения параметров;
3. воспроизводить их по seed;
4. независимо оптимизировать каждый старт;
5. выбрать минимальный взвешенный RSS внутри одной топологии;
6. передать лучший внутримодельный результат общему селектору.

В `FitResult` добавлены `starts_attempted`, `starts_succeeded` и `best_start_index`. CLI принимает `--restarts` и `--restart-seed`. По умолчанию остаётся один старт: новый режим не включён безусловно до оценки затрат на более широком корпусе.

## Результат

| Метрика | Один старт | Четыре старта |
|---|---:|---:|
| Точная двухдуговая топология | `2/10` | `2/10` |
| Медианное время на спектр | `1,01 с` | `3,54 с` |
| Суммарное время | `24,57 с` | `51,88 с` |
| Все четыре старта сошлись | — | `10/10` файлов |
| Победил не первый старт | — | `5/10` файлов |

Multi-start существенно улучшил внутренний fit истинной двухдуговой схемы в пяти файлах. В четырёх из них эффект был крупным:

- RSS `0,00592 → 0,00490`;
- RSS `0,11849 → 0,00630`;
- RSS `0,15839 → 0,00477`;
- RSS `0,10274 → 0,00637`.

В последнем случае диагностический статус истинной модели улучшился с `BAD` до `WARN`.

## Почему точность топологии не выросла

Multi-start решает локальную оптимизацию, но не создаёт информацию, отсутствующую в спектре. После улучшения RSS многие истинные двухдуговые модели всё равно имеют широкие доверительные интервалы, коррелированные параметры, статус `BAD` и конкурирующую простую модель с сопоставимым или лучшим BIC.

Текущий надёжностный селектор правильно не выбирает `BAD` только потому, что мы как авторы генератора знаем скрытую истину. В реальном эксперименте такая рекомендация была бы ложной уверенностью.

Точная топология и надёжная физическая рекомендация — разные метрики:

- **статистически лучший fit** отвечает, какая модель минимизирует критерий;
- **рекомендуемая модель** отвечает, какой результат достаточно устойчив для интерпретации;
- **истинная схема генератора** показывает, была ли информация о топологии восстановима при данном шуме и частотном окне.

## Вывод

Multi-start следует сохранить как управляемый режим и использовать для сложных кандидатов или повторного анализа `WARN/BAD`, но пока не включать четыре старта для всех 17 схем по умолчанию.

Следующий эксперимент должен менять не селектор, а информативность задачи:

1. сетка разделения времён: 1,0 / 1,5 / 2,0 / 3,0 декады;
2. шум: 0 / 0,2 / 0,5 / 1 / 2 / 5%;
3. частотное окно и число точек;
4. несколько стартов только для моделей, прошедших быстрый screening;
5. отдельный вывод `best_statistical` и `recommended_reliable`.
