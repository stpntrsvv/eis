# Единый inference engine

## Зачем понадобился оркестратор

Отдельные BIC, bootstrap, profile likelihood и DRT не должны конкурировать за роль одного магического числа. `eis_inference.py` сохраняет их независимость, но собирает единое решение:

```text
recommended
models_indistinguishable
insufficient_information
analysis_failed
```

Статистический победитель и надёжная рекомендация являются разными полями. Поэтому программа может показать лучший fit, но отказаться рекомендовать его физически.

## Два режима

`fast` сохраняет прежнее поведение одиночного EIS:

```text
загрузка → Lin-KK → multi-start → BIC/статус
```

`reliable` добавляет:

```text
topology bootstrap → DRT stability → optional calibrated resolution map
```

Отсутствие manifest или resolution-карты не ломает быстрый анализ. Тяжёлые проверки являются надстройкой.

## Первый единый вердикт

Для LG MJ1 SOC 50%:

- Lin-KK: `PASS`;
- лучший статистический fit: последовательная двухдуговая CPE-схема;
- fit: `WARN`;
- topology bootstrap: устойчивого победителя нет;
- высокочастотная DRT-область устойчива;
- низкочастотная область неустойчива;
- частотный предел синтетически достаточен;
- информационный пробел локализован в повторяемости или модели области `0,01–0,1 Гц`.

Поэтому `recommended_reliable = null`, а следующий эксперимент указан явно. [Полный отчёт](../../../validation_data/reports/2026-07-17-unified-inference-lg-soc50.md).

## Система критериев версии 1

На этом набор критериев считается собранным. Следующая работа относится не к изобретению новых метрик, а к:

1. интеграции контракта в batch CLI и GUI;
2. калибровке порогов на более широком синтетическом корпусе;
3. ускорению режима `reliable`;
4. замораживанию JSON-схемы;
5. переходу к продуктовому интерфейсу и SPICE/C-экспорту.
