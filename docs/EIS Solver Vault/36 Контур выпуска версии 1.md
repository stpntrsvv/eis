# Контур выпуска версии 1

## Текущее обозначение

Текущий код имеет версию `0.9.0` и статус кандидата на версию 1. Номер
`1.0.0` разрешён только после прохождения всех выпускных ворот ниже.

Это обозначение относится к самостоятельному EIS Solver. Оно не означает
готовность всей будущей Chem Suite.

## Что входит в версию 1

### Рабочий контур

- загрузка TXT, CSV, DAT, BioLogic MPT и MPR;
- выбор канала импеданса, очистка точек и запись метаданных разбора;
- проверка Крамерса — Кронига;
- единый автоматический подбор `adaptive_v2` в GUI и CLI;
- явные наборы схем и ручная схема с границами параметров;
- пакетная обработка;
- диаграммы Найквиста и Боде, остатки и таблицы параметров;
- статусы `OK`, `WARN`, `BAD`, BIC и диагностические флаги;
- экспорт CSV, XLSX, отчёта и изображений;
- атомарный проверенный пакет SPICE;
- атомарный пакет C для контроллера в вариантах `float32` и Q31.

### Исследовательские возможности

Эти функции можно использовать для исследования, но версия 1 не должна
выдавать их результат за откалиброванную вероятность истины:

- bootstrap и профиль правдоподобия параметров;
- точная рекомендация `W`, `Wo` или `Ws`;
- DRT как доказательство конкретного химического процесса;
- совместный подбор SOC-серии и сглаживание траекторий;
- интервалы с признаком `calibrated=true`;
- автоматическое присвоение электрическим элементам химических названий.

Они не удаляются из кода, но их ограничения должны оставаться видимыми.

### Не входит

- модули циклирования, CV, OCV, GITT/PITT;
- облачная библиотека пресетов;
- автоматическое управление измерительным прибором;
- гарантия работы на любом неизвестном формате или приборе;
- промышленная готовность C-модели для STM32F103 без проверки на железе.

## Единый вычислительный путь

`fit_spectrum()` в `eis_pipeline.py` является общей точкой подбора:

```text
CLI ─┐
     ├─> fit_spectrum() ─> adaptive_v2 или явный список ─> FitResult
GUI ─┘
```

Автоматический режим передаёт `circuits=None` и включает два уровня
маршрутизации. Выбранные наборы и ручной ввод передают явный список и никогда
не получают скрытых дополнительных схем. Общий слой также отвечает за
бюджеты, начальные зерна, второй уровень по остаткам и отмену между схемами.

## Автоматическая приёмка

Главная команда:

```powershell
.\.venv\Scripts\python.exe eis_release_check.py `
  --require-spice `
  --require-gcc `
  --require-packaged `
  --packaged-exe dist\eis_qt\eis_qt.exe `
  --ngspice D:\path\to\ngspice_con.exe `
  --gcc C:\path\to\gcc.exe
```

Она создаёт строгий паспорт JSON и проверяет:

1. полный набор автоматических тестов;
2. создание GUI без экрана;
3. функциональный запуск собранного exe на обычном текстовом спектре;
4. функциональный запуск exe на BioLogic MPT через `galvani`;
5. настоящий анализ командной строкой;
6. создание контроллерного пакета;
7. строгую сборку обоих вариантов C;
8. создание SPICE-пакета и настоящий запуск ngspice.

Временные пакеты удаляются после проверки. Любое обязательное нарушение даёт
ненулевой код завершения.

## Сборка

Воспроизводимый набор прямых зависимостей записан в
`requirements-lock.txt`. `requirements.txt` остаётся незакреплённым списком
для разработки.

```powershell
.\.venv\Scripts\python.exe -m PyInstaller --clean --noconfirm eis_app.spec
```

Список сторонних лицензий кладётся внутрь каталога программы. `galvani`
добавлен в сборку явно, поскольку его отложенный импорт не был обнаружен
первоначальной конфигурацией PyInstaller.

Скрытый ключ собранной программы:

```powershell
dist\eis_qt\eis_qt.exe --release-smoke "путь\к\спектру"
```

Он создаёт GUI, читает настоящий спектр, выполняет KK и один реальный подбор,
после чего возвращает код `0`, `1` или `2`.

## Лицензионное решение владельца

Сборка включает `galvani` под GPL-3.0-or-later. PySide6 также имеет условия
LGPL/GPL или коммерческого лицензирования. Проект пока не содержит выбранной
лицензии владельца.

Перед распространением владелец должен явно выбрать один путь:

1. открытое распространение на совместимых условиях;
2. необходимые коммерческие лицензии;
3. замена или исключение несовместимых зависимостей.

Это выпускное решение нельзя принимать автоматически как техническую
деталь. Текущий кандидат разрешён для внутренней проверки.

## Проверка на чистой машине

На отдельной Windows-машине без Python:

1. скопировать весь каталог `dist\eis_qt`, а не только exe;
2. проверить наличие `THIRD_PARTY_NOTICES.md`;
3. запустить программу обычным способом;
4. открыть TXT/CSV и BioLogic MPT/MPR;
5. выполнить автоматический подбор и экспорт CSV/XLSX;
6. выполнить скрытую проверку на заранее переданном контрольном спектре;
7. проверить SPICE с явно выбранным ngspice;
8. записать версию Windows, код завершения и SHA-256 exe.

Только после этой независимой проверки можно менять `0.9.0` на `1.0.0`.
