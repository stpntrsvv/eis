# Глоссарий EIS Solver

Эта страница — общий словарь проекта. Английские названия сохранены там, где они являются именами методов, элементов схемы, полей программы или общепринятыми обозначениями. В обычном тексте документации предпочтительны русские пояснения.

## Основные понятия

| Термин | Значение |
|---|---|
| **EIS** | Электрохимическая импедансная спектроскопия: измерение отклика системы на малый переменный сигнал в диапазоне частот. |
| **Импеданс, `Z`** | Комплексное сопротивление переменному току. Состоит из действительной части `Re(Z)` и мнимой части `Im(Z)`. |
| **Спектр** | Набор значений частоты и соответствующего комплексного импеданса. |
| **Эквивалентная схема** | Электрическая модель, составленная из сопротивлений, ёмкостей и других элементов для приближённого описания измеренной системы. |
| **Фитинг** | Численный подбор параметров модели, при котором её отклик становится как можно ближе к экспериментальным данным. |
| **Остаток, residual** | Разность между экспериментальным и рассчитанным импедансом в конкретной точке. График остатков помогает увидеть систематическую ошибку, скрытую за красивой кривой. |
| **Невязка** | Численная мера общего расхождения модели и данных. В проекте используется взвешенная невязка. |
| **Начальное приближение** | Значения параметров, с которых оптимизатор начинает поиск. Неудачное приближение может привести к плохому локальному решению. |
| **Границы параметра, bounds** | Разрешённый диапазон значения параметра. Границы удерживают поиск в численно и физически разумной области. |
| **Идентифицируемость** | Возможность однозначно определить параметры по имеющимся данным. Красивое совпадение кривых ещё не означает, что каждый параметр найден надёжно. |
| **Переобученная модель** | Слишком сложная схема, которая хорошо повторяет конкретный шумный спектр, но не даёт устойчивого физического объяснения. |
| **Пакетный анализ, batch** | Последовательная обработка нескольких файлов по одним правилам. |
| **Конвейер анализа** | Полная цепочка: чтение файла, очистка данных, проверка Крамерса—Кронига, фитинг, сравнение моделей и экспорт. |

## Графики и диапазоны частот

| Термин | Значение |
|---|---|
| **Диаграмма Найквиста, Nyquist plot** | График `-Im(Z)` против `Re(Z)`. Частота на осях явно не показана, поэтому направление спектра нужно учитывать отдельно. |
| **Диаграмма Боде, Bode plot** | Графики модуля `|Z|` и фазы импеданса в зависимости от частоты. |
| **Высокочастотная область** | Часть спектра на больших частотах. Часто сильнее отражает последовательное сопротивление, контакты, проводку и быстрые процессы. |
| **Низкочастотная область** | Часть спектра на малых частотах. Часто связана с медленным переносом вещества, поляризацией и дрейфом системы. |
| **Дуга** | Полукруглый или приплюснутый участок на диаграмме Найквиста, обычно соответствующий характерному времени процесса. |
| **Диффузионный хвост** | Низкочастотный участок, который может быть связан с переносом вещества и описываться элементами Варбурга. |
| **Индуктивная петля** | Участок спектра с индуктивным характером. Может отражать физический процесс либо артефакты проводки и измерительной установки. |

## Элементы эквивалентной схемы

| Обозначение | Значение |
|---|---|
| **`R`** | Сопротивление, Ом. `R0` обычно означает последовательное или омическое сопротивление системы. |
| **`C`** | Идеальная ёмкость, Ф. Подходит для близкого к идеальному ёмкостного отклика. |
| **`CPE`** | Элемент постоянной фазы. Описывает распределённый, неидеальный ёмкостный отклик шероховатой или неоднородной границы. |
| **`Q`** | Масштабный параметр CPE. Он не всегда равен обычной ёмкости и зависит от показателя `α`. |
| **`α`** | Показатель CPE. При `α → 1` элемент приближается к идеальной ёмкости; уменьшение `α` означает более распределённый отклик. |
| **`W`** | Полубесконечный элемент Варбурга для диффузионного отклика. |
| **`Wo`** | Конечный Варбург с открытой границей. |
| **`Ws`** | Конечный Варбург с короткозамкнутой границей. |
| **`L`** | Индуктивность, Гн. В EIS может описывать как реальную петлю, так и артефакт измерительной цепи. |
| **`p(A,B)`** | Параллельное соединение элементов `A` и `B` в синтаксисе `impedance.py`. Дефис между элементами означает последовательное соединение. |
| **`Rct`** | Сопротивление переноса заряда. В конкретной строке схемы может называться `R1`, `R2` и так далее. |

## Проверка данных и выбор модели

| Термин | Значение |
|---|---|
| **Крамерс—Крониг** | Соотношения, связывающие действительную и мнимую части линейного, причинного и стационарного отклика. Используются как проверка согласованности данных. |
| **Lin-KK** | Практическая линейная проверка Крамерса—Кронига через реконструкцию спектра распределением RC-звеньев. |
| **`mu`** | Критерий Шёнлебера, применяемый при выборе сложности Lin-KK-реконструкции. Это диагностический показатель, а не вероятность правильности данных. |
| **RMSE** | Среднеквадратичная ошибка. В проверке Lin-KK характеризует среднее расхождение реконструкции и измеренного спектра. |
| **RSS** | Сумма квадратов остатков. В проекте для сравнения моделей используется взвешенный вариант. |
| **AIC** | Информационный критерий Акаике: сопоставляет качество приближения со сложностью модели. Меньшее значение предпочтительнее при сравнении моделей на одном наборе данных. |
| **BIC** | Байесовский информационный критерий. Штрафует лишние параметры сильнее AIC; сейчас служит главным критерием выбора среди допустимых моделей. |
| **`PASS`** | Проверка данных не обнаружила существенного нарушения по текущим порогам. Это не доказательство правильности выбранной схемы. |
| **`WARN`** | Есть повод внимательно проверить данные, параметры или остатки, но результат ещё может быть полезен. |
| **`FAIL`** | Проверка согласованности данных выявила серьёзную проблему. Физическую интерпретацию фитинга нужно отложить до выяснения причины. |
| **`OK`** | У результата фитинга нет предупреждений по текущим диагностическим правилам. |
| **`BAD`** | Результат фитинга имеет тяжёлые признаки ненадёжности: невозможные параметры, попадание в границы, неидентифицируемость или другую критическую проблему. |
| **Флаг** | Короткое диагностическое сообщение о конкретной проблеме результата. Флаги нужно читать вместе с остатками и физическим смыслом параметров. |
| **Bootstrap, бутстрэп** | Повторный fit псевдоданных, полученных случайной перевыборкой остатков или наблюдений. Показывает воспроизводимость оценки без доверия только локальной ковариации оптимизатора. |
| **Profile likelihood, профиль правдоподобия** | Проверка параметра, при которой его последовательно фиксируют, а остальные параметры каждый раз переоптимизируют. Показывает ширину и форму допустимой долины решений. |
| **Доверительный интервал** | Диапазон оценок, построенный процедурой с заданным частотным покрытием. Не является вероятностью того, что конкретный истинный параметр лежит внутри уже вычисленного интервала. |

## Файлы, библиотеки и интерфейсы

| Термин | Значение |
|---|---|
| **`.mpr`** | Бинарный формат BioLogic. В проекте читается через `galvani.BioLogic.MPRfile`; загрузка и фитинг проверены на реальных лабораторных данных. |
| **`.mpt`** | Текстовый экспорт BioLogic. Читается через `galvani` с запасными вариантами кодировки. |
| **SmartStat** | Один из поддерживаемых источников текстовых таблиц EIS. Если специальные признаки формата не найдены, применяется универсальный текстовый разбор. |
| **`galvani`** | Библиотека для чтения файлов BioLogic. Её лицензия GPL требует отдельного внимания при закрытом коммерческом распространении. |
| **`impedance.py`** | Библиотека, которая предоставляет `CustomCircuit`, элементы схем и реализацию Lin-KK. |
| **CLI** | Интерфейс командной строки, реализованный в `eis_cli.py`. Удобен для автоматизации и воспроизводимых проверок. |
| **GUI** | Графический интерфейс пользователя. Активная реализация находится в `eis_qt.py` и использует PySide6/Qt. |
| **Pro mode** | Расширенный режим GUI с ручными схемами, семействами моделей, начальными значениями, границами и локальными пресетами. |
| **Пресет** | Сохранённый набор ручной схемы и её параметров. Пользовательские пресеты хранятся локально и не входят в репозиторий. |
| **Метаданные** | Сведения об источнике, формате, выбранном канале, колонках и способе разбора файла. Нужны для воспроизводимости анализа. |
| **Экспортный контракт** | Стабильный состав и смысл колонок в CSV/XLSX. Подписи интерфейса можно переводить, но программные имена полей должны оставаться постоянными. |
| **SPICE** | Семейство программ и форматов для расчёта электрических схем. В проекте внешняя проверка выполняется в ngspice, но она подтверждает инженерное воспроизведение импеданса, а не истинность электрохимической интерпретации. |
| **Сеть Фостера** | Последовательная сеть из параллельных RC-секций, а также возможных последовательных R и L. Используется как пассивное инженерное представление рациональной модели. |
| **Список соединений, netlist** | Текстовое описание элементов схемы, их номиналов и соединений. Пользовательский SPICE-экспорт сохраняется в `model.lib`. |
| **Паспорт инженерной модели** | Строгий JSON-файл с источником, научной ECM, рабочей полосой, порогами, ошибками, выбранными секциями, версией симулятора и контрольными суммами. |
| **Атомарная публикация** | Выдача результата целиком одним завершённым пакетом. Если проверка или запись прерывается, конечный каталог модели не появляется. |
| **`float32`** | 32-разрядное представление числа с плавающей точкой. В контроллерном пакете этот вариант принимает ток в амперах и возвращает напряжение в вольтах. |
| **Фиксированная точка** | Представление дробного числа целым кодом с заранее заданным положением двоичной точки. Оно позволяет считать без аппаратного блока плавающей точки. |
| **Q31, Q1.31** | Знаковый 32-разрядный формат фиксированной точки: один знаковый разряд и 31 дробный. Диапазон — от `-1` до значения чуть меньше `+1`. |
| **Полный масштаб** | Физическая величина, соответствующая границе нормированного числового диапазона. В пакете задаётся полный масштаб тока, а полный масштаб напряжения вычисляется из сопротивления постоянному току. |
| **Период расчёта** | Время между последовательными вызовами дискретной модели. Вместе с полосой оно определяет ошибку переноса непрерывных RC-секций в контроллер. |
| **Эталонные векторы** | Таблица заранее рассчитанных входов и выходов, по которой проверяется перенос модели в другую среду или на контроллер. |
| **Smoke test, дымовой тест** | Короткая проверка основного рабочего сценария: программа запускается, данные читаются, анализ завершается, результат появляется. |
| **Регрессионный тест** | Автоматическая проверка того, что ранее работавшее поведение не сломалось после изменения кода. |
